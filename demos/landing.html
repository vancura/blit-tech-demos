Features

Core Rendering

  • WebGPU dual-pipeline architecture
  • Pixel-perfect integer coordinates
  • Primitives: pixels, lines, rectangles
  • Sprite rendering with batching
  • Bitmap font system

Game Features

  • Fixed 60 FPS timestep
  • Camera scrolling system
  • Color tinting and effects
  • Asset loading and caching
  • Clean BT namespace API

Getting Started

Installation

npm install blit-tech

Quick Example

import { BlitTech, BT, Color32, Vector2i } from 'blit-tech';

class MyGame {
    queryHardware() {
        return {
            displaySize: new Vector2i(320, 240),
            targetFPS: 60,
        };
    }

    async initialize() {
        // Load assets here
        return true;
    }

    update() {
        // Game logic (60 FPS)
    }

    render() {
        BT.clear(Color32.black());
        BT.drawRectFill(
            new Rect2i(100, 100, 50, 50),
            Color32.white()
        );
    }
}

const blitTech = new BlitTech();
await blitTech.run(new MyGame());

Requirements

  • Chrome/Edge 113+ (Windows, macOS, Linux, Android)
  • Firefox Nightly (with WebGPU enabled)
  • Safari 18+ (macOS/iOS)
  • Compatible GPU and drivers

Inspiration

Blit-Tech is heavily inspired by RetroBlit, a retro pixel game framework for Unity by Martin Cietwierkowski. RetroBlit provides an ideal environment for pixel-perfect retro games through a traditional game loop and code-only development.

Blit-Tech brings this philosophy to the web using WebGPU: no scene graphs, no complex frameworks â€“ just sprites, primitives, and fonts.
